CocoDB is a django app to manage player data for a scrabble organisation.

The tech stack is django and postgres, and it will be deployed via docker using dokku.

The main tables are:

1. Players:
- player_number : string
- name : string

2. Ratings:
- player_id : fk into Players
- rating: int
- last_played: date

The user interface should be a text box and a button labelled "find player"
with a live search-as-you-type on the player name, and when a player is
selected clicking the button should populate a box below the text field with
the players's details (join the player and ratings tables to display all the
information we have about the player).

If javascript is not enabled this should fall back to searching on the backend
and returning a page with the same data populated.

Use https://www.cocoscrabble.org/ as a template for design choices - display
the logo on top of the page, and use the same button styling.
